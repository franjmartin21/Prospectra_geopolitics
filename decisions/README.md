# CEO Decisions

Every significant decision made by the CEO is logged here as a standalone file.

## File naming convention
`YYYY-MM-DD_<decision-slug>.md`

## What gets a decision file
- Changes to the investment thesis or framework
- Curriculum sequencing changes (skipping, reordering, adding topics)
- Databricks architecture pivots
- Any call to pause, accelerate, or redirect the project
- Responses to major world events that change the strategic direction

## What does NOT need a decision file
- Routine lesson delivery
- Weekly briefings
- Minor Databricks task adjustments

## Decision file format
Each file should contain:
- **Date and trigger** — what prompted this decision
- **Decision** — what was decided, in plain language
- **Reasoning** — why, including alternatives considered
- **Implications** — what changes as a result
- **Owner** — who executes (usually Bolo)
